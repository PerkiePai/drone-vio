  What this tells you

  Speed вҖ” LightGlue wins decisively and consistently: 17вҖ“51 ms vs SuperGlue's steady ~110вҖ“124 ms, i.e. 2.5вҖ“6Г— faster. Note LightGlue's adaptive behavior is
  visible: the hard pair (0вҶ’3, lots to match) costs it 51 ms, but the easy/sparse pairs (0вҶ’1, 0вҶ’12) drop to 17вҖ“23 ms as it prunes and early-exits. SuperGlue's
  cost is flat regardless вҖ” it can't skip work.

  Quality вҖ” comparable, slight edge to LightGlue: inlier ratios track each other closely (within a few points). LightGlue produces more matches and keeps a
  similar-or-better inlier fraction at short baselines.

  Pose agreement (sanity check): both recover near-identical motion вҖ” ~1В° at 1s, rising to ~26вҖ“34В° by 12s. They agree, which validates the matches. The 0вҶ’1
  SuperGlue 0.0В° is a degenerate near-planar recoverPose result, not a real difference.

  Baseline degradation: both fall off fast as the gap grows (352вҶ’20 matches for SG; 505вҶ’35 for LG over 1sвҶ’12s). For VIO this confirms you want to match
  consecutive frames, not skip them.

  One honest caveat on "fair"

  The input pixels and keypoint budget are equalized, but the detection thresholds aren't: SuperGlue's SuperPoint (threshold 0.005) returns ~600 keypoints,
  while LightGlue's returns the full 1024 (lower default threshold). So LightGlue starts with more candidates вҖ” part of why it gets more matches. To make it a
  pure matcher comparison I should set LightGlue's SuperPoint detection_threshold=0.005 so both detect the same ~600 points.
